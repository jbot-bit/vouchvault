# Telegram Privacy Policy

## Key Privacy Principles

Telegram operates on two foundational commitments: "We don't use your data to show you ads" and "We only store the data that Telegram needs to function as a secure and feature-rich messaging service."

## Data Collection Overview

**Account Information**: Users provide phone numbers and basic profile data (name, picture, about section) to create accounts. Screen names, profile pictures, and usernames remain public by design. Telegram does not require real names.

**Optional Profile Data**: Birthday information can be added with granular privacy controls. Telegram Business subscribers may add fixed locations and business hours to their profiles.

**Contact Syncing**: With permission, Telegram stores contacts (phone numbers and names only) to notify users when contacts join and display names in notifications.

**Messages and Media**:
- Cloud chats store messages, photos, videos, and documents on encrypted servers across multiple jurisdictions
- Secret chats use end-to-end encryption with no server storage; "There is **no way** for us or anybody else without direct access to your device to learn what content is being sent"
- Public channels and groups are cloud chats with encryption in storage and transit

**Location Data**: Shared locations are treated as regular messages. Live location sharing and "Make Myself Visible" features allow location access while apps are closed.

## Security Measures

Data stored by users in the EEA is kept in Netherlands data centers. All data uses heavy encryption, with keys stored separately across multiple jurisdictions. End-to-end encrypted data (secret chats, calls, Telegram Passport) is encrypted before reaching servers, rendering it unintelligible to Telegram.

## Data Sharing

Personal data may be shared with:
- Other Telegram users you communicate with
- Group companies (Telegram Group Inc., Telegraph Inc., Telegram FZ-LLC)
- Law enforcement with valid judicial orders
- Google or Microsoft for translation/transcription services (audio data only for voice-to-text)

Bots receive basic public account data and any data shared during interaction.

## User Rights and Deletion

Users can request data copies, deletion, amendments, or processing restrictions. Accounts can be deleted via a deactivation page, removing all cloud data permanently. Inactive accounts self-destruct after 18 months by default (adjustable in settings). Messages in secret chats delete on both devices; cloud chat messages require both parties to delete separately unless using the new one-on-one chat feature allowing either party to delete all history for both sides without time limits.
